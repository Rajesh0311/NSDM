# NSDM Institute Website Restructure Brief

## Purpose

This brief defines the next major restructure of the NSDM public website.

The goal is to evolve the site from a research-programme landing page into the public front door for the emerging:

NSDM Institute for Decision Boundaries

The site must remain accurate, conservative, and evidence-backed, while becoming clearer, more professional, more useful, and more visually coherent.

---

## Core Positioning

NSDM Institute for Decision Boundaries

Research, benchmarks, methods, and tools for justified AI decisions.

Core thesis:

AI should not only answer.  
It should know whether it is justified.

Expanded statement:

NSDM builds benchmarks, methods, papers, and decision-evaluation tools for distinguishing supported decisions from contradicted, under-specified, governance-ambiguous, sovereign-risk, and reward-aligned but unjustified actions.

---

## Why the Restructure Is Needed

The current site contains strong research material, but its flow is still too internal.

It explains the taxonomy and benchmark, but it does not yet clearly show:

- what NSDM is
- who should use it
- what they can download
- how it helps their work
- why the research matters in practice
- how the programme evolves into a serious institute
- how the sovereign stack and benevolent systems branches fit into the long-term roadmap

The new site should answer those questions quickly.

## What Is NSDM?

Suggested copy:

NSDM — Neuro-Symbolic Decision Boundaries — evaluates whether an AI output or action is justified by the available evidence, rules, claim, action context, governance authority, and reward pressure.

Most AI evaluation asks whether the answer is correct.

NSDM asks whether the answer is justified.

That distinction matters because an AI system can produce a fluent, useful, or reward-aligned output that is still unsafe to act on.

## Latest Benchmark Evidence

Keep this section concise and evidence-backed.

Suggested copy:

A zero-shot LLM baseline performed strongly on conventional support and contradiction, but had zero mean recall on ambiguous governance and reward-aligned unjustified action.

Few-shot prompting improved mean accuracy from 0.703 to 0.753, but the hardest NSDM-specific labels remained difficult across repeated runs.

This shows why aggregate accuracy is not enough.

### Result Table

| Evaluation | Mean Accuracy |
|---|---:|
| Clean modular boundary reasoner | 0.733 |
| Evidence-state reasoner v0.2 | 0.733 |
| Zero-shot LLM, five-run mean | 0.703 |
| Few-shot LLM, five-run mean | 0.753 |

## Sovereign Stack Boundaries

This is the NSDM-S branch.

Suggested copy:

AI decisions are not justified only because the output appears useful.

The stack itself must be justified: model choice, data exposure, vendor dependence, token cost, infrastructure control, routing logic, memory, auditability, and governance authority.

NSDM-S extends decision-boundary reasoning from model outputs to the enterprise AI stack.

### Use Cases

- AI stack audits
- Token cost justification
- Vendor dependence scoring
- Data exposure reviews
- Model routing governance
- AI sovereignty readiness
- Decision boundary evidence ledger

---

## Benevolent System Boundaries

This is the NSDM-B branch.

Suggested copy:

A powerful AI system should not merely obey.

It should preserve autonomy, reduce regret, avoid obedient harm, and know when to ask, refuse, escalate, or defer to human judgement.

NSDM-B studies the boundary between helpful action, harmful obedience, paternalism, and autonomy-preserving support.

### Use Cases

- Agent safety
- Human autonomy protection
- Regret-sensitive recommendations
- Escalation boundaries
- Paternalism detection
- Power asymmetry governance

## Visual Direction

The site should feel like:

A serious, futuristic research institute for accountable AI decision-making.

Not a blog.  
Not a startup hype page.  
Not a raw GitHub README.

Visual rules:

- Deep navy / blue-black background
- Subtle grid and mathematical linework
- Glassmorphism cards
- Cyan, teal, violet, green, amber, and magenta accents
- One consistent card system
- One consistent data-card system
- One consistent download-card system
- Clear section spacing
- No generic robot, chatbot, or stock AI imagery
