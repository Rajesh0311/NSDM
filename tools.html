Decision-Boundary Utilities

Tools for justified AI decisions.

Most AI readiness tools ask whether an organisation can adopt AI. NSDM asks a harder question: can the organisation make AI decisions that are justified, governed, contestable, and safe to act on?

View tools Back to institute
NSDM test
  • Is the claim supported?
  • Is a premise missing?
  • Does evidence contradict the claim?
  • Is authority ambiguous?
  • Is reward pressure distorting the action?
  • Should the system answer, abstain, refuse, or escalate?

Tool layer

These utilities translate NSDM research into practical checks for executives, AI builders, governance teams, and product owners. They are designed to expose decision risk before a model output becomes an operational action.

01

AI Readiness Assessment

Evaluates whether an organisation is ready to use AI in decision workflows, not only whether it has data, cloud access, or automation ambition.

  • Decision ownership
  • Evidence availability
  • Governance maturity
  • Human override path
02

Use-Case Scorer

Ranks AI use cases by value, risk, evidence quality, governance clarity, and actionability.

  • Business value
  • Decision risk
  • Data readiness
  • Boundary complexity
03

RAG Readiness / Data-Rot Audit

Checks whether retrieval systems are relying on current, authoritative, and decision-relevant sources.

  • Source authority
  • Document freshness
  • Contradictory sources
  • Evidence traceability
04

Decision-Boundary Checklist

Tests whether a model output is supported, under-specified, unsupported, contradicted, or governance-ambiguous.

  • Supported claim
  • Missing premise
  • Contradiction
  • Escalation trigger
05

TokenOps Cost Card

Connects token spend to justified decisions rather than raw usage volume.

  • Cost per justified decision
  • Cost per safe refusal
  • Negative-value token rate
  • Boundary stress-test cost
06

Governance Credibility Test

Assesses whether a proposed AI workflow has clear authority, policy coverage, auditability, and human accountability.

  • Policy authority
  • Role permission
  • Audit trail
  • Review threshold
Exportable decision pack

From model output to defensible decision.

NSDM tools are designed to produce a compact decision pack: the claim, evidence, governing rule, boundary state, action recommendation, and escalation logic.

Evidence state supported, unsupported, contradicted, under-specified, ambiguous governance, or reward-aligned unjustified.
Governance state policy allowed, policy forbidden, authority missing, human review required, audit required.
Action state answer, ask for clarification, request more evidence, abstain, refuse, escalate, or block.
TokenOps value whether the tokens spent produced a justified decision, a safe refusal, or avoidable noise.
Research status: These tools are part of an active research and product programme. Early GOV-0 experiments suggest that text-only embeddings can recognise topic structure, but struggle with evidence-state boundaries. Structured NSDM diagnostic features substantially improved performance on a targeted contrast set. The current tools should be treated as decision-support utilities, not automated certification.
